### 4.2 多租户数据隔离策略

采用**共享数据库、独立Schema**的混合模式：

1. **租户标识**: 所有业务表包含 `tenant_id` 字段
2. **查询过滤**: 中间件自动添加租户过滤条件
3. **数据权限**: 用户只能访问所属租户的数据
4. **超级管理员**: 可跨租户查看数据

## 6. 安全设计

### 6.1 认证授权

- **JWT Token**: Access Token (15分钟) + Refresh Token (7天)
- **权限控制**: RBAC模型，支持菜单、按钮、数据权限
- **API限流**: 基于用户/IP的速率限制

### 6.2 数据安全

- **密码加密**: bcrypt算法
- **敏感数据**: AES加密存储
- **SQL注入**: ORM参数化查询
- **XSS防护**: 输入过滤、输出转义

## 7. 部署架构

### 7.1 Docker Compose配置

```yaml
version: '3.8'
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
  
  frontend:
    build: ./frontend
    
  backend:
    build: ./backend
    environment:
      - DATABASE_URL=mysql://...
      - REDIS_URL=redis://...
  
  celery-worker:
    build: ./backend
    command: celery -A config worker -l info
  
  celery-beat:
    build: ./backend
    command: celery -A config beat -l info
  
  mysql:
    image: mysql:8.0
    volumes:
      - mysql_data:/var/lib/mysql
  
  redis:
    image: redis:7-alpine
```

### 7.2 生产环境建议

- **负载均衡**: Nginx / ALB
- **数据库**: 主从复制 + 读写分离
- **缓存**: Redis Cluster
- **文件存储**: MinIO / 阿里云OSS
- **监控**: Prometheus + Grafana
- **日志**: ELK Stack
